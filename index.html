CAP — Confidence-Adjusted Probability v1.0
Purpose: Shrinks raw Poisson P(over) toward 50% to account for real-world uncertainty not captured by the projection formula. Feeds EV% in the Book Lines box. Does not change tier badges — those remain edge-based until validated against forward tracking data.

Formula:
p1 = 0.50 + (raw_p − 0.50) × (1 − σ_shrink)
p2 = 0.50 + (p1 − 0.50) × (1 − sample_penalty)
CAP = 0.50 + (p2 − 0.50) × (1 − stability_penalty)
σ shrinkage = min(35%, σ × 8%) per unit of standard deviation
  σ 1.0 → 8% shrink  ·  σ 1.5 → 12%  ·  σ 1.8 → 14%  ·  σ 2.4 → 19%

Sample penalty = uncertainty from small start samples
  10+ starts → 0%  ·  7–9 → 4%  ·  5–6 → 8%  ·  4 → 14%

Stability penalty = divergence between L10 avg and K/9 projection
  <1.0K gap → 0%  ·  1.0–2.0K → 4%  ·  >2.0K → 8%
Example — Misiorowski June 6
Proj 10.7K · O8.5 · Raw P(over) = 74%
σ 1.8 → σ_shrink 14% → p1 = 71.6%
12 starts → sample_penalty 0% → p2 = 71.6%
L10 9.0 vs proj 10.7 → gap 1.7K → stability 4% → CAP = 70%

Book est O8.5 = +100 (implied 50%). EV using CAP 70%: +32%
EV using raw 74%: +48% — CAP gives the more honest number.
CAP is displayed on each card as "CAP XX% · P(O) YY%". The Book Lines EV% uses CAP, not raw P(O).
Threshold for future tier upgrades: if CAP-Strong ≥65% hit rate over 60+ tracked results, tiers will be redefined using CAP rather than edge size.
